Duality.Resources Namespace Duality Documentation
 
Classes
  Class Description
AbstractShader
Represents an OpenGL Shader in an abstract form.
AudioData
Stores compressed audio data (Ogg Vorbis) in system memory as well as a reference to the OpenAL buffer containing actual PCM data, once set up. The OpenAL buffer is set up lazy i.e. as soon as demanded by accessing the AlBuffer property or calling SetupAlBuffer.
BatchInfo
BatchInfos describe how an object, represented by a set of vertices, looks like.
DrawTechnique
DrawTechniques represent the method by which a set of colors, Textures and vertex data is applied to screen.
Font
Represents a font. While any system font or imported TrueType font can be used, they are internally pre-rasterized and stored in a Texture with an [!:Duality.Resources.Texture.Atlas].
FragmentShader
Represents an OpenGL FragmentShader.
Material
Materials are standardized BatchInfos, stored as a Resource. Just like BatchInfo objects, they describe how an object, represented by a set of vertices, looks like. Using Materials is generally more performant than using BatchInfos but not always reasonable, for example when there is a single, unique GameObject with a special appearance: This is a typical BatchInfo case.
Pixmap
A Pixmap stores pixel data in system memory.
Pixmap Layer
Represents a pixel data layer.
Prefab
Prefab is short for "prefabricated object" and encapsulates a single GameObject that can serve as a template. When creating a GameObject out of a Prefab, it maintains a connection to it using a PrefabLink object. This ensures that changes made to the Prefab propagate to all of its instances as well. It also keeps track of Properties that have been deliberately modified in the editor and restores them after re-applying the original Prefabs data.
PrefabLink
Represents a GameObjects connection to the Prefab it has been instanciated from.
RenderTarget
Instead of rendering to screen, RenderTargets can serve as an alternative drawing surface for a Camera. The image is applied to one or several Textures. By default, only the first attached Texture is actually used, but you can use a custom FragmentShader to use all available Textures for storing information.
Scene
A Scene encapsulates an organized set of GameObjects and provides update-, rendering- and maintenance functionality. In Duality, there is always exactly one Scene Current which represents a level, gamestate or a combination of both, depending on you own design.
ShaderProgram
Represents an OpenGL ShaderProgram which consists of a Vertex- and a FragmentShader
Sound
A Sound is parameterized AudioData. Note that a Sound Resource does not contain any kind of audio data by itsself.
Texture
A Texture refers to pixel data stored in video memory
VertexShader
Represents an OpenGL VertexShader.
