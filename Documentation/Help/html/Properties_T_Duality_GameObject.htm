GameObject Properties Duality Documentation
The GameObject type exposes the following members.
Properties
  NameDescription
Active
[GET / SET] Whether or not the GameObject is currently active. To return true, both the GameObject itsself and all of its parent GameObjects need to be active.
ActiveSingle
[GET / SET] Whether or not the GameObject is currently active. Unlike Active, this property ignores parent activation states and depends only on this single GameObject. The scene graph and other Duality instances usually check Active, not ActiveSingle.
AffectedByPrefabLink
[GET] The PrefabLink that connects this object or one or its parent GameObjects to a Prefab.
Camera
[GET] The GameObject's Camera Component, if existing.
ChildCount
[GET] The number of child GameObjects this object has.
Children
[GET] Enumerates this objects child GameObjects.
ChildrenDeep
[GET] Enumerates all GameObjects that are directly or indirectly parented to this object, i.e. its children, grandchildren, etc.
ComponentCount
[GET] The number of Components this object consists of.
Disposed
[GET] Returns whether this GameObject has been disposed. Disposed GameObjects are not to be used and should be treated specifically or as null references by your code.
FullName
[GET] The path-like hierarchial name of this GameObject.
HierarchyLevel
[GET] Returns the number of parents this object has when travelling upwards the scene graph hierarchy.
Id
[GET] The GameObjects persistent globally unique identifier.
Name
[GET / SET] The name of this GameObject.
Parent
[GET / SET] This GameObject's parent object in the scene graph. A GameObject usually depends on its parent in some way, such as being positioned relative to it when occupying a position in space.
PrefabLink
[GET] The PrefabLink that connects this object to a Prefab.
Renderer
[GET] The GameObject's Renderer Component, if existing.
RigidBody
[GET] The GameObject's RigidBody Component, if existing.
Transform
[GET] The GameObject's Transform Component, if existing.
