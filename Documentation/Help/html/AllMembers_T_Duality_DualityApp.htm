DualityApp Members Duality Documentation

The DualityApp type exposes the following members.
Methods
  NameDescription
CheckOpenALErrors
Checks for errors that might have occured during audio processing.
CheckOpenGLErrors
Checks for errors that might have occured during video processing. You should avoid calling this method due to performance reasons. Only use it on suspect.
DisposeLater
Schedules the specified object for disposal. It is guaranteed to be disposed by the end of the current update cycle.
GetAvailDualityTypes
Enumerates all available Duality Types that are assignable to the specified Type.
GetDualityAssemblies
Enumerates all currently loaded assemblies that are part of Duality, i.e. Duality itsself and all loaded plugins.
GuardSingleThreadState
Guards the calling method agains being called from a thread that is not the main thread. Use this only at critical code segments that are likely to be called from somewhere else than the main thread but aren't allowed to.
Init
Initializes this DualityApp. Should be called before performing any operations withing Duality.
LoadAppData
Triggers Duality to (re)load its DualityAppData.
LoadMetaData
Triggers Duality to (re)load its DualityMetaData.
LoadSaveAll
Loads all Resources that are located in this DualityApp's data directory and saves them again. All loaded content is discarded both before and after this operation. You usually don't need this.
LoadUserData
Triggers Duality to (re)load its DualityUserData.
Render
Performs a single render cycle.
SaveAppData
Triggers Duality to save its DualityAppData.
SaveMetaData
Triggers Duality to save its DualityMetaData.
SaveUserData
Triggers Duality to save its DualityUserData.
Terminate
Terminates this DualityApp. This does not end the current Process, but it isn't recommended to attemp performing any Duality operations after it has been terminated.
Update
Performs a single update cycle.

Properties
  NameDescription
AppData
[GET / SET] Provides access to Duality's current application data. This is never null. Any kind of data change event is fired as soon as you re-assign this property. Be sure to do that after changing its data.
AppDataPath
[GET] Returns the path where this DualityApp's application data is located at.
AvailableModes
[GET] Enumerates all available GraphicsModes.
DefaultMode
[GET] Returns the GraphicsMode that Duality intends to use by default.
DisposedPlugins
[GET] Enumerates all plugin assemblies that have been loaded before, but have been discarded due to a runtime plugin reload operation. This is usually only the case when being executed from withing the editor or manually triggering a plugin reload. However, this is normally unnecessary.
ExecContext
[GET] Returns the DualityApp ExecutionContext in which this DualityApp is currently running.
ExecEnvironment
[GET] Returns the DualityApp ExecutionEnvironment in which this DualityApp is currently running.
Keyboard
[GET] Provides access to keyboard user input
LoadedPlugins
[GET] Enumerates all currently loaded plugins.
MetaData
[GET] Provides access to Duality's current meta data. This is never null.
MetaDataPath
[GET] Returns the path where this DualityApp's meta data is located at.
Mouse
[GET] Provides access to mouse user input.
Sound
[GET] Provides access to the main SoundDevice.
TargetMode
[GET / SET] The GraphicsMode in which rendering takes place. Setting this will not actually change Duality's state - this is a pure "for your information" property.
TargetResolution
[GET / SET] The size of the current rendering surface (full screen, a single window, etc.) in pixels. Setting this will not actually change Duality's state - this is a pure "for your information" property.
UserData
[GET / SET] Provides access to Duality's current user data. This is never null. Any kind of data change event is fired as soon as you re-assign this property. Be sure to do that after changing its data.
UserDataPath
[GET] Returns the path where this DualityApp's user data is located at.
